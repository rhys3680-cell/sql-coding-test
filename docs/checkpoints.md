# 점검 트리거 (Checkpoints)

학습의 페이스를 유지하고 "얕은 반복"을 방지하기 위한 **자동 점검 규칙**.

## 트리거

### 1. 3문제마다 (빈도 체크)
연속 3문제 풀이 후 반드시 **미니 점검**.

**할 것** (3분):
- 방금 푼 3문제의 **공통 개념**을 한 문장으로 말해보기
- 막혔던 곳이 있다면 왜 막혔는지 태그화 (`stuck_on` 필드)
- 피곤한가? 집중력 측정 — 떨어졌으면 세션 종료

### 2. 토픽 전환 시 (깊이 체크)
새 주제로 넘어가기 전 **반드시**.

**할 것** (5~10분):
- 지금 토픽에서 배운 **모든 문법** 나열 (문서 보지 말고)
- 각 문법이 **언제 쓰이는지** 한 줄 설명
- 못 나열한 게 있으면 **그 주제 아직 안 끝남**
- 세션 리포트 생성 (`log.py stats --session`)

### 3. 세션 종료 시 (리포트)
**할 것**:
- 누적 지표 확인 (`log.py stats`)
- 다음 세션 예고 (어떤 토픽, 대략 몇 문제)
- 재도전 대상 플래그 (오답·gave_up 문제)

## 점검에서 나오는 산출물

점검 결과는 다음 액션으로 연결되어야 한다:

| 관찰 | 다음 액션 |
|------|-----------|
| 같은 `stuck_on`이 2번 이상 | 그 지점 별도 미니 세션 |
| 정답률 100% (easy만) | 난이도 상향 |
| 정답률 < 60% | 개념 다시 학습 (앞으로 가지 말 것) |
| 3일 이상 공백 | 다음 세션은 "복습부터" |

## 안 하면 생기는 일

- 방법론 문서에만 쓰여 있고 실제론 안 지켜짐 → 지난 번 발생
- **트리거 없이 "점검해야지" 라고만 생각하면 절대 안 함**
- 이 문서의 목적: 점검을 **자동 조건**으로 고정
